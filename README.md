# Project Overview

## Project Name

Self Running Quest App
Working Title: **RunQuest PH**

## Project Concept

RunQuest PH is a gamified jogging and running management app designed for local users in the Philippines.

The app helps users build a running habit by turning real-world jogging, walking, and marathon training into a game-like adventure. Users complete running courses, earn points, gain experience, level up, unlock new routes, and continue their fitness journey as if they are exploring a fantasy world.

The core idea is to combine:

* Real-world running
* Game-style progression
* Adventure and exploration
* Level-up rewards
* Local Philippine running courses
* English-first user experience

## Target Users

The primary target users are local residents in the Philippines who want to exercise regularly but need motivation, structure, and entertainment.

Main user groups may include:

1. Beginners who want to start jogging
2. Office workers who want to improve health
3. Students and young adults who like games
4. Casual runners preparing for fun runs or marathons
5. Local communities that want group challenges
6. Fitness-conscious users who prefer simple and fun tracking

## Main Problem

Many people want to exercise but fail to continue because jogging can feel repetitive, boring, and lonely.

Traditional fitness apps usually focus on numbers such as distance, pace, calories, and time. However, many beginners are not motivated by numbers alone.

This app solves the problem by making exercise feel like a game. Users do not simply “run 3 kilometers.” They “clear a course,” “gain experience,” “level up,” and “unlock the next adventure.”

## Main Goal

The main goal of the app is to help users build a consistent running habit through game-style motivation.

The app should make users feel:

* “I want to run again.”
* “I want to complete the next course.”
* “I want to level up.”
* “I want to unlock the next area.”
* “I want to compare my progress with friends.”

## Core Gameplay Idea

The user selects a running course provided by the app.

After completing the course, the user receives rewards such as:

* Points
* Experience
* Level progress
* Badges
* Course completion records
* Possible virtual items
* Access to new courses

As the user completes more routes, the user’s character or profile becomes stronger.

The system may work like an RPG-style adventure game, similar in feeling to games such as Lineage or Diablo, but applied to real-world jogging and running.

## Basic User Flow

1. User creates an account.
2. User chooses a starting level or beginner mode.
3. App recommends a nearby running course.
4. User starts running.
5. App tracks distance, time, and course completion.
6. User completes the course.
7. App gives points and experience.
8. User levels up after reaching enough experience.
9. App unlocks new courses, missions, or achievements.
10. User continues the journey.

## Language

The default language of the app is English.

Future language support may include:

* Filipino / Tagalog
* Korean
* Japanese
* Chinese

However, the first version should focus on simple and clear English for local Philippine users.

## Location Focus

The first market is the Philippines.

The app should consider local conditions such as:

* Hot weather
* Rainy season
* Safety of running routes
* Urban traffic
* Parks, malls, and safe jogging areas
* Local fun run culture
* Mobile-first usage
* Affordable or free access expectations

## Initial Product Direction

The first version should not try to become a full fitness platform immediately.

The MVP should focus on:

1. User registration
2. Course list
3. Course detail
4. Start run
5. Record distance and time
6. Complete course
7. Earn points
8. Level up
9. Show user profile and progress
10. Admin can create and manage courses

Suggested identity keywords:

* Adventure
* Quest
* Level up
* Exploration
* Challenge
* Local routes
* Habit building
* Beginner-friendly
* Community
* Fun running

## Development Principle

The project should be developed step by step.

Phase 0 should focus on concept and planning.
Phase 1 should create the basic app/server structure.
Phase 2 should create user and course data models.
Phase 3 should implement running course completion.
Phase 4 should implement points and level system.
Phase 5 should implement admin course management.
Phase 6 should improve game features and user retention.

The first goal is not to build a perfect app, but to create a working MVP that proves users can select a course, run, complete it, earn points, and level up.
