# Project Overview

## Project Name

Working Title: **RunQuest PH**

Alternative names:

* RunQuest Philippines
* JogQuest PH
* Island Runner PH
* Run Adventure PH
* FitQuest Philippines

The working title for now is **RunQuest PH**.

---

## 1. Project Concept

RunQuest PH is a mobile-first gamified walking and jogging app for local users in the Philippines.

The app helps users build a consistent exercise habit by turning real-world walking, jogging, and running into a game-style journey. Users complete real courses in selected Philippine areas, earn points, gain experience, level up, and unlock new routes and missions.

The app starts with a **Philippine local travel style** and can later expand into a **fantasy RPG style**, where each real Philippine location becomes a fantasy-inspired adventure area.

The core concept is:

> Walk. Jog. Run. Explore the Philippines. Level up your journey.

---

## 2. First Product Goal

The first goal of the app is a combination of:

1. **Personal exercise habit management**
2. **Local running community building**

The app should first help individuals continue walking and jogging regularly. After that, it can expand into a community platform where users compare progress, join challenges, and participate in local running events.

The first version should not try to become a complete fitness, social, event, and commerce platform at once. The first goal is to prove that users enjoy the basic loop:

> Choose a course → Walk or jog → Complete the course → Earn XP → Level up → Unlock the next journey

---

## 3. Target Users

The primary users are local residents in the Philippines.

Main target groups include:

1. Beginners who want to start walking or jogging
2. Office workers who need a simple exercise habit
3. Young adults who enjoy games and progress systems
4. Casual runners who join fun runs or local running events
5. People living or working near BGC, Makati, Ayala Triangle, and MOA
6. Users who want a more enjoyable alternative to traditional fitness tracking apps

The app should be beginner-friendly. It should not feel like an app only for serious marathon runners.

---

## 4. Initial Location Focus

The first service area should focus on selected Metro Manila running-friendly locations.

Initial areas:

1. **BGC**
2. **Ayala Triangle / Makati**
3. **MOA / Pasay**
4. Possible future expansion to other Metro Manila areas

These areas are suitable because they already have walking, jogging, and running culture. Some areas such as Makati and Ayala Avenue also have regular Sunday morning running activities. BGC, Makati, and MOA are also known for public events, fun runs, and active urban lifestyles.

The first version should not cover the whole Philippines. It should start from selected high-quality routes where users can actually walk or run safely.

---

## 5. Exercise Type

The first version will support:

1. Walking
2. Jogging
3. Running

The app should not be positioned only as a marathon app in the beginning.

The first message should feel accessible:

> You do not need to be a runner. Start with walking. Then jog. Then run. Your journey grows with you.

This makes the app suitable for beginners and casual users.

---

## 6. Game Style

The initial game style is:

> Philippine local travel adventure

Users feel like they are exploring different places in the Philippines by completing walking and jogging courses.

Later, the app can expand into:

> Fantasy RPG based on Philippine locations

For example:

* BGC can become a modern city fortress
* Makati can become a golden business kingdom
* MOA / Manila Bay can become a seaside adventure zone
* Cebu can become an island warrior route
* Palawan can become a hidden paradise quest
* Baguio can become a mountain endurance zone

The game style should begin with familiar real-world locations and slowly add fantasy elements.

---

## 7. Platform Strategy

The first version should be developed as a:

> Mobile Web App / PWA

Reasons:

1. Lower development cost
2. Faster MVP testing
3. Easy sharing through links
4. Easier updates compared to native apps
5. Suitable for early-stage user testing in the Philippines

Future versions may expand into:

1. Android app
2. iOS app
3. Watch integration
4. Native GPS background tracking

However, the MVP should start with a mobile web app or PWA.

---

## 8. GPS and Activity Verification

The first version should support up to **Level 2 GPS tracking**.

### Level 1

User manually starts and completes a course.

### Level 2

The app uses GPS to track distance, time, and basic route completion.

### Future Level 3

The app verifies whether the user followed a specific route correctly.

In the future, the app may accept data from wearable devices or smart watches. Users who own a watch may be able to upload or sync running data for more accurate activity verification.

The first version should not depend on watch integration. Watch data should be treated as a future premium or advanced feature.

---

## 9. Monetization Strategy

The first version should be free.

Initial model:

1. Free app

Future model:

1. Premium subscription
2. Monthly subscription target: PHP 100 to PHP 200
3. Premium missions
4. Advanced stats
5. Special badges
6. Ad-free experience
7. Premium fantasy maps
8. Watch data integration
9. Personalized training plans

Long-term monetization possibilities:

1. Brand partnerships
2. Corporate wellness programs
3. Running events
4. Sponsored routes
5. Local merchant coupons
6. Fitness and sports brand promotions

However, the main business goal after the free version is:

> Premium subscription at around PHP 100–200 per month

---

## 10. Core Gameplay Loop

The core loop is:

1. User selects a course.
2. User starts walking, jogging, or running.
3. App tracks activity.
4. User completes the course.
5. User earns XP and points.
6. User levels up.
7. New courses, badges, or missions are unlocked.
8. User returns to complete the next journey.

The app should make exercise feel like progress in a game.

Traditional fitness app message:

> You ran 3.2 km.

RunQuest PH style message:

> Quest Completed!
> You cleared Ayala Sunday Run.
> You earned 350 XP.
> You reached Level 3: City Explorer.
> Next route unlocked: BGC Morning Loop.

## 13. Product Identity

RunQuest PH should not feel like a serious medical fitness app.

It should feel:

* Fun
* Local
* Motivating
* Game-like
* Beginner-friendly
* Filipino-user friendly
* Mobile-first
* Adventure-oriented
* Community-friendly

Suggested slogan ideas:

1. Walk. Jog. Run. Level up.
2. Explore the Philippines one run at a time.
3. Turn every run into a quest.
4. Your fitness journey becomes an adventure.
5. Run your city. Unlock your journey.

## 14. Development Direction

The project should be developed step by step.

Recommended phase structure:

### Phase 0

Planning documents and product concept

### Phase 1

Basic mobile web app and backend setup

### Phase 2

User account and profile

### Phase 3

Course database and admin course management

### Phase 4

Start / complete activity tracking

### Phase 5

GPS distance and time tracking

### Phase 6

XP, points, and level system

### Phase 7

Badges and basic achievements

### Phase 8

PWA installation and mobile optimization

### Phase 9

Advertisement structure

### Phase 10

Premium subscription planning

The first technical milestone is:

> A user can open the mobile web app, select a course, start walking or jogging, complete the course, earn XP, and level up.
